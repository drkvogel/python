>// is unconditionally "flooring division", e.g:
```py
>>> 4.0//1.5
2.0
```
>As you see, even though both operands are floats, `//` *still* floors -- so you always know securely what it's gonna do.

>>> def f():
...     print('func')
...     print(self.__name__)

>>> def f():
...     print('func')
...     print(__name__)
>>> f()

__main__
